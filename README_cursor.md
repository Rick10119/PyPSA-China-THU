# PIPS China 供热-电力解耦开发任务书

## 1. 项目背景与目标

### 现状
- 现有代码库：PIPS China，包含热力需求与电力系统的耦合模型
- 当前模型在电力系统规划运行中同时决策：供电技术选择 + 供热技术选择 + 供热 dispatch
- 数据：包含 8760 小时（全年逐时）的省级电力系统数据

### 目标（更新：弃用旧电价方案）
历史原因，原先“外部导入电价/电价模拟方案”的描述与实现已经弃用（电价口径不真实）。本仓库现在采用 **基于已求解 PyPSA 网络结果的事后电价重构**：

1. **输入**：容量规划/出清求解完成后的网络 `postnetwork-*.nc`（含 `n.generators_t.p`、`n.generators.p_nom_opt` 等）
2. **算法**：先按省份本地火电出力的“当周负荷率”映射本地报价曲线；再在省间联络线**未拥塞**且存在**净送电**时，用送端省份的本地映射价按线路 `efficiency` 折算输电损失，对受端省份形成可竞争的进口报价，并与本地价取更优（默认取更小，即 `import_agg: min_offer`）。
3. **输出**：逐时逐省电价 `pd.DataFrame(index=snapshots, columns=provinces)`，用于后续分析/绘图

### Task 2: 电价重构接口设计（替代旧方案）

#### 输入
- solved PyPSA network：`postnetwork-*.nc`（或内存中的 `pypsa.Network` 对象）

#### 输出
- `reconstruct_market_prices(n)` 返回逐时逐省电价表（省级电力 AC bus）

#### 实现位置
- 代码：`scripts/reconstruct_market_prices.py`
- 主要思想：对每个省份计算
  \[
  LR(t)=P_{thermal}(t) / \max(P_{thermal})\_\mathrm{week}
  \]
  再将 \(LR\) 代入分段线性报价曲线得到电价。

#### 两阶段：规划 + 分段火电 dispatch 与电价口径（可选）

在容量规划得到 `postnetwork-*.nc` 之后，可启用 `config.yaml` 中的 `dispatch_segmented_prices`：将装机固定为 `p_nom_opt`、投资成本置零，仅对配置中的煤电/气电主通道（如 `coal cc` 的 `Generator`、`OCGT gas` 的 `Link`）用并联容量块近似**分段报价**，再跑一次 8760 经济调度。Snakemake 规则：`run_dispatch_segmented` → `export_dispatch_segmented_prices`。

**与事后 `reconstruct_market_prices` 的关系（避免重复加价）**：

- `scripts/export_reconstructed_prices.py --price-mode mapped`（默认）：使用周负荷率映射曲线，适用于**规划阶段**解出的 `postnetwork`，此时优化里火电多为单一边际成本。
- 对**分段 dispatch 解出的网络**，应使用 `--price-mode marginal`：直接使用各省级 AC bus 的 `buses_t.marginal_price`（不做省间进口折算；输电已在 dispatch 对偶中体现）。**不要**再对同一结果跑 `mapped` 模式，否则等于在已内生分段的出清之上再叠一层负荷率报价映射。

**`mapped` 模式下“跨省输电”考虑逻辑（当前实现已开启）**

- 本地价：先对每个省按本地火电出力构造 `local_price(p, t)`（负荷率映射曲线）。
- 联络线筛选：仅处理 `bus0` 与 `bus1` 都是省级电力节点（省-省 link）的线路。
- 未拥塞判定：当 `|p0| <= p_nom - line_cong_eps_mw` 视为可用外来电通道。
- 潮流方向：
  - `p0 > min_inflow_mw`：表示 `bus0 -> bus1` 净送电，受端 `bus1` 可获得来自 `bus0` 的进口报价；
  - `p0 < -min_inflow_mw`：表示 `bus1 -> bus0` 净送电，受端 `bus0` 可获得来自 `bus1` 的进口报价。
- 进口报价折算：送端本地价按线路效率折算损耗，`offer = local_price(send) / efficiency`（反向用 `efficiency2`，缺失则回落到 `efficiency`）。
- 省内最终价：把本地价与所有可行进口报价聚合后取优：
  - `import_agg: min_offer`（默认）取最小进口报价，再与本地价取 `min`；
  - `import_agg: max_offer` 取最大进口报价，再与本地价取 `min`。
- 参数位置：`scripts/export_reconstructed_prices.py` 的 CLI 参数 `--import-agg`、`--line-cong-eps-mw`、`--min-inflow-mw`。

#### 分段 `marginal_cost` 与 `shares`：全成本报价与五段模板（`dispatch_segmented_prices`）

**成本边界（现货重构 / 二阶段 dispatch）**  
二阶段分段里每一段 `marginal_cost`（EUR/MWh，电能量）应按**全可变报价**填报：**燃料 + 碳履约机会成本 + 变动运维（VOM）**。当 `zero_gas_fuel_marginal_cost: true` 时，底层不再用气/煤燃料 Generator 的动态燃料价，并联块上的 **`marginal_cost` 即为机组在市场中申报的绝对段价**，须覆盖该口径。

**与 `data/costs/costs_{year}.csv` 的关系**：成本表单行推得的「仅燃料+VOM」（见下表左侧）常与**单体 LMP**同量级（例：求解网络山东约 **34** EUR/MWh）；若要做与**国内现货保本线**对齐的分段标价，应采用下面「基准 SRMC（市场口径）」行（右侧），二者可并存：**规划解用成本表**，**分段标尺用 EUR 全成本数组**。

| 口径 | 煤电（EUR/MWh_e，约） | 气电 OCGT（EUR/MWh_e，约） | 说明 |
|------|------------------------|----------------------------|------|
| 本仓库 `costs_2025.csv` 单段（燃料+VOM，见 `coal` / `gas`+`OCGT` 行） | **34**（13.93/η+VOM） | **~104**（气价/η+链接 VOM，未拆双层计价时） | 与未显式加计碳、`costs` 中气价 proxy 同源 |

### Task 3: （可选）供热优化模型重构

#### 数学模型调整
**原模型（耦合）**：
```
min Σ(电力运营成本 + 供热运营成本)
s.t. 电力平衡约束
     热力平衡约束  
     CHP 电-热耦合约束
     ...
```

**新模型（解耦）**：
```
min Σ(供热投资成本 + 供热运营成本 + 购电成本)
其中：购电成本 = Σ_{node,h}( 外部边际电价_{node,h} × 虚拟购电量_{node,h} )
s.t. 热力平衡约束（逐时）
     供热技术运行约束
     供热技术容量约束
     CHP 电-热耦合约束（供热相关，保留）
     电侧简化平衡：虚拟购电量 = 热侧电力消耗（热泵/电锅炉/CHP 辅机等）在该电节点的用电
     （可选）不显式建电网物理约束；仅用外生边际电价完成成本结算
```

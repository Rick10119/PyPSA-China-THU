# PIPS China 供热-电力解耦开发任务书

## 1. 项目背景与目标

### 现状
- 现有代码库：PIPS China，包含热力需求与电力系统的耦合模型
- 当前模型在电力系统规划运行中同时决策：供电技术选择 + 供热技术选择 + 供热 dispatch
- 数据：包含 8760 小时（全年逐时）的省级电力系统数据

### 目标
将供热模块与电力系统**解耦**，而且只保留天津相关的节点，改为：
1. **输入**：从外部已优化完的电力系统模型读取固定电价（或从 Excel 读取预计算结果）
2. **决策**：仅保留供热技术选择与供热 dispatch 优化
3. **输出**：供热决策方案 + 供热曲线

### Task 3: 供热优化模型重构

#### 数学模型调整
**原模型（耦合）**：
```
min Σ(电力运营成本 + 供热运营成本)
s.t. 电力平衡约束
     热力平衡约束  
     CHP 电-热耦合约束
     ...
```

**新模型（解耦）**：
```
min Σ(供热投资成本 + 供热运营成本 + 购电成本)
其中：购电成本 = Σ_{node,h}( 外部边际电价_{node,h} × 虚拟购电量_{node,h} )
s.t. 热力平衡约束（逐时）
     供热技术运行约束
     供热技术容量约束
     CHP 电-热耦合约束（供热相关，保留）
     电侧简化平衡：虚拟购电量 = 热侧电力消耗（热泵/电锅炉/CHP 辅机等）在该电节点的用电
     （可选）不显式建电网物理约束；仅用外生边际电价完成成本结算
```

#### 代码修改点
- [ ] 核心优化：删除/屏蔽“电力系统扩建/调度/输电约束”相关变量与约束，但**保留 CHP 电-热耦合**与所有供热侧约束
- [ ] 电价输入：新增参数 `electricity_prices[node, hour]`（或 `province, hour`）作为外生边际电价
- [ ] 电侧接口：在每个电节点添加“虚拟发电机/电网购电源”，其 `marginal_cost`（可为逐时序列）=外部导入边际电价；容量给足够大以不成为约束
- [ ] 成本核算：目标函数只包含供热相关投资/运行成本 + 虚拟购电成本（由外生边际电价计算）
- [ ] 保留变量：供热技术容量、供热 dispatch、储热状态、CHP 相关出力（电/热）等

## 6. 注意事项

1. **保留 git 历史**：建议新建 branch `heat-decoupled`，不要直接修改 main
2. **备份原模型**：确保随时可以对比原耦合模型的结果
3. **CHP 处理**：CHP **不能移除**；凡与供热相关的 CHP 设备、变量与电-热耦合约束都应保留（否则供热可行域会被错误改变）
4. **电价来源**：电价采用**外部导入的边际电价**（按节点/省份逐时），用于虚拟购电源的边际成本；需明确其单位与口径（元/MWh 或 EUR/MWh，是否含税/网费等）
5. **时间对齐**：确保 8760 小时与原有热力需求数据的时区/季节对齐
